# SEO

## Overview

SEO is implemented through metadata, structured data, generated files
(sitemap, robots.txt, RSS feed), and content strategy. Every page has a
unique title, description, canonical URL, and OpenGraph/Twitter card.

---

## Metadata Strategy

### Per-Page Metadata

All pages define metadata using `pageMetadata()` from `src/lib/seo.ts`:

```typescript
export const metadata = pageMetadata({
  title: "Projects — React, Next.js & DevOps Case Studies",
  description: "Case studies of production React and Next.js websites and platforms...",
  path: "/projects",
});
```

The helper generates:

- `<title>` with site name suffix (except home page).
- `<meta name="description">`.
- `<link rel="canonical">`.
- `<meta property="og:*">` — title, description, URL, site name, type.
- `<meta name="twitter:*">` — card (summary_large_image), title, description.
- `robots` meta tag (index/follow by default, disabled on 404 pages).

### Default Metadata

The root layout (`layout.tsx`) exports `defaultMetadata` which provides:

- **metadataBase** — `new URL(SITE_URL)` for relative URL resolution.
- **Icons** — site logo as favicon (via ImageKit).
- **Home page metadata** — title focused on long-tail keywords:
  "React/Next.js Developer & Community Builder in Nairobi, Kenya".

### Blog Post Metadata

Blog post pages generate metadata dynamically from Hashnode data:

```typescript
export async function generateMetadata({ params }) {
  return {
    title: `${post.title} — ${SITE_NAME}`,
    description: post.brief,
    keywords: post.tags.map((t) => t.name),
    alternates: { canonical: url },
    openGraph: { type: "article", publishedTime, authors, tags, images },
    twitter: { card: "summary_large_image", title, description, images },
  };
}
```

- Keywords are derived naturally from the post's own tags.
- OpenGraph `article` type with `publishedTime` and `authors`.
- Cover image used as OG/Twitter image when available.

## Structured Data (JSON-LD)

Three schema types are rendered across the site:

### Person Schema

Rendered once in the root layout:

```json
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Lawrence Juma",
  "alternateName": "jumalaw98",
  "url": "https://jumalaw98.vercel.app",
  "jobTitle": "React/Next.js Developer",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "Nairobi",
    "addressCountry": "KE"
  }
}
```

### BreadcrumbList Schema

Rendered on every sub-page:

| Page       | Breadcrumb                |
| ---------- | ------------------------- |
| About      | Home → About              |
| Projects   | Home → Projects           |
| Case study | Home → Projects → [title] |
| Blog       | Home → Blog               |
| Blog post  | Home → Blog → [title]     |
| Community  | Home → Community          |
| Contact    | Home → Contact            |

### Article Schema

Rendered on blog post pages:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "...",
  "description": "...",
  "datePublished": "...",
  "author": { "@type": "Person", "name": "..." },
  "publisher": { "@type": "Person", "name": "Lawrence Juma" }
}
```

## Generated Files

### Sitemap (`/sitemap.xml`)

Dynamic sitemap generated by `src/app/sitemap.ts`. Includes:

- All static routes: `/`, `/projects`, `/about`, `/contact`, `/community`,
  `/blog`.
- All project case study slugs (`/projects/[slug]`).
- All blog post slugs (`/blog/[slug]`) from Hashnode or placeholders.

Each entry has a `lastModified` date. Blog entries use the post's
`publishedAt` date. Placeholder posts are only included when Hashnode is
not configured.

### Robots.txt (`/robots.txt`)

Generated by `src/app/robots.ts`:

```text
User-agent: *
Allow: /
Sitemap: https://jumalaw98.vercel.app/sitemap.xml
```

### RSS Feed (`/blog/rss.xml`)

Valid RSS 2.0 feed with:

- Post title, link, GUID, publication date, and description (brief) per item.
- Sorted newest-first.
- Declared as alternate link in blog page metadata:
  `<link rel="alternate" type="application/rss+xml" href="/blog/rss.xml">`.
- 1-hour cache.

### OpenGraph Image (`/opengraph-image.png`)

Generated fallback OG image at request time via `next/og` (Satori):

- 1200×630 PNG.
- Brand-blue background with orange circle accent + "JLW" monogram.
- Site name + tagline as text.
- Used automatically as the fallback OG/Twitter image for any page that
  does not pass an explicit `ogImage` to `pageMetadata()`.

## Content Strategy

### Long-Tail Keywords

Target keywords baked into default metadata and page content:

| Keyword                              | Page            |
| ------------------------------------ | --------------- |
| "React Next.js developer Nairobi"    | Home, About     |
| "Kenya frontend developer portfolio" | Home            |
| "DevOps community organizer Kenya"   | Home, Community |
| "React/Next.js case studies Kenya"   | Projects        |

### Heading Hierarchy

- Pages use a single `<h1>` followed by `<h2>` sections.
- Visually-hidden `<h2>` elements are used where needed to maintain a valid
  document outline without visual disruption:
  - Projects listing: `<h2 class="sr-only">All Case Studies</h2>`.
  - Blog listing: `<h2 class="sr-only">Filtered Articles</h2>`.

### Semantic HTML

- `<nav>` with `aria-label` for navigation landmarks.
- `<article>` for blog posts and case studies.
- `<section>` for page sections.
- `<header>` for page/article headers.
- `<aside>` for sidebar content (table of contents).
- `aria-current="page"` on active nav links.

## Blog Short Links

Deterministic short codes (`/s/<code>`) resolve to full blog post URLs:

- Generated from slug using DJB2 hash algorithm (`src/lib/shortId.ts`).
- Same slug always produces the same short code — no storage required.
- Redirects (307) to `/blog/[slug]`.
- ISR with 1-hour revalidation.

## Why Not a CMS

- Hashnode provides the blogging platform with built-in SEO features.
- Static content (projects, community, about) lives in TypeScript data files —
  no database, no CMS, no build-time content pipeline to maintain.
- The RSS integration means blog content is decoupled from the site's deploy
  cycle — publish on Hashnode, appear on the site within an hour.
